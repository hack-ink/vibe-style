# Decisions Index

Purpose: Route agents to durable rationale, tradeoffs, and consequences that should
survive implementation churn.

Question this index answers: "why is it shaped this way?"

## Use this index when

- You need the rationale behind a repository policy, architecture boundary, or durable
  tradeoff.
- You are deciding whether a new change contradicts an existing decision.

## Do not use this index when

- You need the authoritative behavior contract; read `docs/spec/index.md`.
- You need the execution procedure; read `docs/runbook/index.md`.
- You need supporting evidence without a durable decision; read `docs/research/index.md`.

## Decision document contract

Start each decision with a compact routing header:

- `Status`
- `Date`
- `Question`
- `Decision`
- `Consequences`

Keep decisions short and link to supporting research when evidence matters.
